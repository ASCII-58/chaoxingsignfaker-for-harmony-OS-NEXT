# Agent Guidance for Chaoxing Sign-in Helper

## Project Overview

- **Type**: HarmonyOS app (ArkTS/ETS)
- **Build system**: Hvigor (not npm)
- **IDE**: DevEco Studio (required for building)

## Build Commands

- Build debug: DevEco Studio → Build → Build Debug Hap
- Clean: DevEco Studio → Build → Clean
- Full rebuild: Delete `entry/build` and rebuild

## Testing

- Unit tests: `entry/src/test/*.ets` (run via DevEco Studio test runner)
- Ohos tests: `entry/src/ohosTest/ets/test/*.ets`
- No CLI test command; tests run through DevEco Studio

## Key Files

- Entry point: `entry/src/main/ets/entryability/EntryAbility.ets`
- Main pages: `entry/src/main/ets/pages/Index.ets`, `LoginPage.ets`, `SignerPage.ets`
- Router config: `entry/src/main/resources/base/profile/router_map.json`
- Module config: `entry/src/main/module.json5`

## Architecture Notes

- Single-module project (entry is the app module)
- Uses @ohos/imageknife for image handling
- Uses @hadss/debug-db for local storage
- Backup ability configured via `EntryBackupAbility.ets`

## Common Errors

- Build fails without HarmonyOS SDK (requires DevEco Studio with SDK)
- `local.properties` is auto-generated by DevEco; do not commit it
- Signing required for installing on device (use test certificate for development)
